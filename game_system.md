# AI育成ゲーム - 転生編 システム仕様書 (Master Specification)

## 1. プロジェクト概要
クォータービューのマップ上でAIキャラクターを育成・観察するシミュレーションゲーム。
HTML5 Canvas + Vanilla JavaScriptによるMVC風アーキテクチャ。
性格診断、全11種族の進化ツリー、「師匠・弟子入りシステム」に加え、新たに「カードバトル(TCG)システム」や「クラウドセーブ連携」を搭載し、大幅に拡張されている。

## 2. ファイル構成と役割 (全15ファイル構成想定)
AIがコードの役割を推論するためのモジュールマップ。

### 2.1 コア・システム系
* `index.html` / `style.css`: UIの土台、各種メニュー、Canvasコンテナ、エフェクト定義。
* `main.js`: エントリポイント。ゲームループの制御、初期化、モード切替、開発・デバッグ用機能（座標微調整など）。
* `system.js`: グローバル変数管理、マップ自動生成、ローカルセーブ/ロード、多言語対応、進化・性格診断のベースロジック。
* `cloud_manager.js` **[NEW]**: クラウドサーバーとの通信、セーブデータのオンライン同期・バックアップ管理。
* `data.js`: マスターデータ群（画像ソース、マップチップ、施設、進化ツリー、性格別セリフ定義）。

### 2.2 ロジック・コントローラー系
* `ai_core.js`: 【中核ロジック】AIのステータス管理、経路探索、タスクキュー実行、師匠クエスト進行管理、作業時の素材自動補充。
* `game_manager.js`: プレイヤーのアクション（農業、建築、売買）、経済システム（借金・破産処理）の管理。
* `tcg_core.js` **[NEW]**: トレーディングカードゲームのコアルール、デッキ管理、カードの効果処理、バトル進行のロジック。
* `ui_controller.js`: DOM操作、パネルUI更新、チャット入力による行動指示判定、演出（進化ルーレット等）の制御。
* `view_renderer.js`: Canvasへのクォータービュー描画、カメラ追従、アニメーション更新。

### 2.3 エンドコンテンツ・拡張系 (別ファイル想定)
* `feature_pokedex.js`: 図鑑機能、発見キャラ閲覧、放牧マップへの配置UI制御。
* `feature_grazing.js`: 【エンドコンテンツ】放牧システムの独立ディレクターロジック、自律作業アニメーション、放置ボーナス計算。

## 3. コアデータ構造 (グローバルステート)
* `assets`: マップ上の全オブジェクト（地形、施設、師匠のキャンプ等）の連想配列。
* `aiPet`: AIキャラの全ステータス（欲求、能力、熟練度）。
  * `apprentice`: 師匠・弟子入り状態（現在の師匠、ランク、課題進捗等）。
  * `learnedWords`: チャットから学習した単語リスト。
* `grazingData`: 放牧マップ（最大5つ）の独立した箱庭状態。
* **[NEW想定]** `tcgData`: 所持カード、現在編成中のデッキ情報、バトル戦績など。

## 4. 主要システムの特徴（AIコンテキスト用）
* **行動決定モデル**: チャット入力（指示）と、性格ベースの自律行動（Idle時の確率発動）のハイブリッド。学習済みの言葉のみタスク化可能。
* **師匠・弟子入りシステム**: フィールドエンカウント → 拠点出現 → ランク1〜9の課題進行。詰み防止のための「素材の裏方自動補充」が機能する。
* **TCGシステム**: （※AIへの指示：`tcg_core.js`のロジックに基づき、NPCとの対戦や収集要素として機能する想定でコードを読むこと）
* **デバッグ機能**: `main.js`等のAI Adjust Modeにより、キャラの座標（`sx`, `sy`, `sw`, `sh`）をリアルタイム調整・出力可能。
